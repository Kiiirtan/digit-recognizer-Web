PIPELINE

Preprocessing

Image Normalization Pipeline

Unlike the interactive canvas, uploaded images vary wildly in size, color, and aspect ratio. The client-side Javascript heavily standardizes these images before sending them to the inference API.

1. Dynamic Inversion

The algorithm checks the corners of the image to determine if it was drawn on dark or light paper. It converts to grayscale, applies a contrast boost, and dynamically inverts the colors to guarantee white ink on a black background.

2. Aspect-Aware Scaling

The digit is strictly cropped to its bounding box. Crucially, instead of stretching the image into a square, it is proportionally scaled down to fit inside a 20x20 pixel bounding box, preserving its natural shape.

3. Tensor Generation

The 20x20 digit is centered perfectly inside a final 28x28 matrix (matching the MNIST standard). The raw pixel intensities are then serialized into JSON and pushed via HTTP POST.

Integration

API Payload Structure

The backend expects a strict JSON schema containing the 2D pixel matrix.
